# `@hubex/design-system`: реестр препятствий к переиспользованию в плагинах

Дата: 2026-07-23. Версия пакета: `0.4.0` (публичный npm).
Аудит с позиции **встраиваемого плагина** FSM-системы HubEx (лёгкий,
tree-shakeable, встраивается в чужой хост, возможно React 19 или вообще Vue).
Все пункты — с доказательствами из распакованного пакета. Документ рассчитан
как конструктивная обратная связь продуктовой команде-владельцу DS.

## Контекст: почему у плагина другие требования

Официальная DS оптимизирована под **основное приложение** (один known-хост,
React 18, styled-components, размер не критичен). Плагин — противоположный
профиль: грузится в **чужой** хост, должен быть минимальным, не должен
конфликтовать с рантаймом/стилями хоста и не может пинить версию React хоста.
Отсюда — расхождение, а не «плохой пакет».

## Реестр

| # | Severity | Проблема | Влияние на плагины |
|---|---|---|---|
| 1 | 🔴 Blocker | `export * as Icons from './icons'` — namespace-реэкспорт **1463** иконок в барреле | Держит весь набор иконок (**5.9 МБ**) в бандле даже при импорте одной кнопки |
| 2 | 🔴 Blocker | Нет `"sideEffects": false`; барредь `export * from` по всем компонентам; единый `index.js` **1.9 МБ** (не сплит) | Tree-shaking фактически не работает — плагин тянет всю DS ради одного компонента |
| 3 | 🔴 Blocker | styled-components@6 как **runtime** (peerDep, 106 обращений к `.theme.`, обязательный `ThemeProvider`) | CSS-in-JS рантайм + риск конфликта нескольких инстансов styled-components в хосте; всё дерево надо оборачивать в провайдер |
| 4 | 🔴 Blocker | `peerDependencies.react: "^18.3.1"` (и `react-dom`) | Исключает React 19 (хост/плагины на 19). Жёсткий conflict при установке |
| 5 | 🟠 High | Нет CSS-only / фреймворк-агностичной сборки (в пакете **ноль** `.css`) | Vue-плагины не могут использовать DS вообще; нельзя взять «только стили» |
| 6 | 🟠 High | `createGlobalStyle` + **8× `@font-face` Roboto** в `GlobalStyles` | Глобальные стили и шрифты протекают в документ хоста; дубли загрузки шрифта |
| 7 | 🟡 Medium | Нет `exports`-map, нет `type:module`; только `main`/`module`/`types` | Нет subpath-импортов (`.../Button`), хрупкое ESM/CJS-резолвинг у потребителя |
| 8 | 🟡 Medium | Runtime-deps `@floating-ui/react`, `@tanstack/react-virtual` | Дополнительный вес и потенциальные дубли у хоста |
| 9 | 🟡 Medium | Токены — только внутри JS-бандла (styled-components theme), не публикуются как артефакт | Чтобы переиспользовать значения, приходится `eval`-ить бандл (что мы и сделали в разведке) |

## Детально

### 1. Namespace-реэкспорт иконок (🔴)
`dist/esm/src/index.d.ts`: `export * as Icons from './icons'`; в `icons/` — 1463
компонента, `5.9 МБ` из `11 МБ` пакета. Namespace-объект нельзя тришейкнуть —
он весь считается используемым. **Фикс:** убрать namespace-реэкспорт; отдавать
иконки только именованными subpath-импортами (`@hubex/design-system/icons/Foo`),
а лучше — icon-font вместо инлайновых SVG-компонентов.

### 2. Tree-shaking выключен де-факто (🔴)
`package.json` без `"sideEffects": false`; JS собран в один `index.js` (1.9 МБ),
`.d.ts` разбиты по компонентам, но код — нет. **Фикс:** `"sideEffects": false`,
пер-компонентные entry-points, `exports`-map с subpath’ами.

### 3. styled-components как рантайм (🔴)
106 обращений `props.theme.*`, обязательный `ThemeProvider`, peerDep
`styled-components@^6.1.11`. В чужом хосте два инстанса styled-components =
рассинхрон темы/классов. **Фикс (идеал):** статическая генерация CSS (нулевой
рантайм) или хотя бы отдельная «zero-runtime» сборка. Реалистично: см. «Минимальный
запрос».

### 4. React 18 pinned (🔴)
`peerDependencies.react: "^18.3.1"`. **Фикс:** расширить до `">=18"` (компоненты
почти наверняка работают на 19). Дешёвый, но разблокирует установку.

### 5. Нет CSS-only сборки (🟠)
Ни одного `.css`. React-only через styled-components. Vue-часть системы отрезана.
**Фикс:** публиковать собранный `.css` (классы) + токены — это делает DS
консумируемой любым фреймворком.

### 6. Протечка глобальных стилей и шрифтов (🟠)
`createGlobalStyle` + 8 `@font-face 'Roboto'` инжектятся в документ хоста.
**Фикс:** сделать `GlobalStyles`/шрифты **opt-in**, скоупить под контейнер DS,
не грузить `@font-face`, если шрифт уже есть у хоста.

### 7–9. Упаковка, deps, непубликуемые токены (🟡)
`exports`-map отсутствует; runtime-deps floating-ui/tanstack-virtual; значения
токенов недоступны иначе как через бандл. **Фикс 9 самый ценный для нас:**
публиковать `tokens` отдельным артефактом (`tokens.json` и/или `variables.css`)
как часть релиза.

## Что реально просить у разработчиков

Разбито по стоимости — от «дёшево и разблокирует нас» до «идеал».

**Минимальный запрос (наибольшая отдача, малая цена для команды):**
1. **Публиковать токены отдельным артефактом** — `tokens.json` + `variables.css`
   в составе релиза (пункт 9). Тогда наш лёгкий `@cvetkov_es/*` официально
   зависит от значений DS как от источника, без всякого рантайма. Это закрывает
   90% нашей боли и не требует трогать компоненты.
2. Расширить `peerDependencies.react` до `">=18"` (пункт 4) — одна строка.

**Средний запрос (оздоровить сам пакет):**
3. `"sideEffects": false` + убрать namespace-реэкспорт иконок + `exports`-map
   (пункты 1, 2, 7). Радикально режет вес и включает tree-shaking.
4. `GlobalStyles`/`@font-face` сделать opt-in и скоупить (пункт 6).

**Идеал (стратегически):**
5. Отдельная **zero-runtime / CSS-only** сборка DS (пункты 3, 5) — тогда её можно
   встраивать и в плагины, и во Vue напрямую, а наш пакет становится тонкой
   обёрткой, а не переизложением.

## Наш путь вне зависимости от ответа

Даже если команда ничего не поменяет: официальный `tokens`-объект **извлекаем**
(проверено в разведке — парсится в JSON без запуска styled-components), поэтому
`@cvetkov_es/*` может брать оттуда значения уже сейчас. Договорённость с командой
лишь делает связь официальной и снимает риск «дрейфа» при их релизах.
